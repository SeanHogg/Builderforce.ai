# Task: Each Key Result has a concrete status with code evidence

## Problem & Goal

- The status of Key Results are not clear or concrete, making it difficult for stakeholders to understand their progress and make data-driven decisions.

## Target users / ICP roles (if relevant)

- Project Manager
- Scrum Master
- Developers and Testers

## Scope

- All Key Results that have been defined and are being worked on during the Product Increment.

## Functional requirements

| Requirement ID | Requirement Name |
| --- | --- |
| KR01 | Each Key Result should have a clear and concrete status. |
| KR02 | The concrete status should be represented as a code evidence. |

## Acceptance criteria

| Requirement ID | Acceptance Criteria |
| --- | --- |
| KR01 | The Product Owner or a designated person can determine the current status of a Key Result by inspecting the code evidence associated with it. |
| KR02 | The code evidence should uniquely identify and support the specific status of a Key Result. |

## Out of scope

- Key Results that have not been defined yet.
- Changes to the definition of Key Result statuses and code evidence without prior approval from the Product Owner or their designated person.
