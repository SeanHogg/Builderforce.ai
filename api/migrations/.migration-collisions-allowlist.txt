# that share a numeric prefix are both applied in a deterministic order — they are
# not broken today. The risk is a future runner that dedupes on the bare prefix
# token and silently skips one. check-migrations.mjs (run in `npm test` / CI) fails
# the build on ANY duplicate numeric prefix so new collisions can never land.

# ledger — a release-level operation, not a CI cleanup. They are accepted-as-is and
# tracked in the README Consolidated Gap Register for a future deliberate backfill.
